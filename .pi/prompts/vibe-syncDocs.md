# Workflow: Documentation Sync

Update `docs/features/` after major code changes.

## Steps

### 1. Identify Changed Components
List the main files changed in the recent work.

### 2. List Existing Feature Docs
```bash
ls docs/features/
```

### 3. Read Existing Docs
Read existing docs to understand current coverage and scope.

### 4. Determine Action
For each changed component decide whether to:
- update an existing doc
- create a new doc
- skip if already covered

### 5. Documentation Standards
Each feature doc should include:
- overview
- architecture
- key components
- optional data flow
- configuration if applicable
- changelog or hotfix notes when useful

### 6. Write Updates
Update or create docs under `docs/features/`.

### 7. Verify Links
Ensure all mentioned file paths are correct.

### 8. Summary
Report what was updated or created.
