# [FR-002] Extension-Owned OAuth Account Management

## Labels
`MUS`, `commands`, `oauth`

## User Story
As a Pi user, I want to manage multiple OAuth identities inside one extension, so that I can operate more than one account for the same upstream.

## Proposed Solution

### Overview
Implement router-specific slash commands for adding, listing, removing, refreshing, enabling, and disabling accounts without relying on Pi’s single-provider OAuth slot.

### Implementation Flow
1. Register router management commands
2. Drive OAuth login through the configured upstream adapter
3. Persist, refresh, and toggle accounts from the extension-owned store

### Technical Approach

```ts
pi.registerCommand("router-login", {
  handler: async (args, ctx) => {
    // add | list | remove | refresh
  },
});
```

### Key Considerations
- Command UX should work without Pi’s built-in `/login` assumptions
- Multiple accounts for the same upstream/provider must coexist
- Commands must avoid revealing secrets in output

## Acceptance Criteria

- [ ] `/router-login add` can add another account without overwriting an existing one
- [ ] `/router-login list` shows multiple saved accounts
- [ ] `/router-login remove <id>` deletes a single targeted account
- [ ] `/router-login refresh <id>` refreshes one targeted account
- [ ] `/router-enable <id>` and `/router-disable <id>` toggle account eligibility
