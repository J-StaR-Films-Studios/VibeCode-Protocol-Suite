# [FR-007] Observability

## Labels
`MUS`, `observability`, `commands`

## User Story
As a Pi user, I want to inspect router status quickly, so that I understand why a given account is or is not being used.

## Proposed Solution

### Overview
Provide status commands and per-account state summaries for enablement, cooldowns, auth health, usage, and failures.

### Implementation Flow
1. Collect merged views from the credential and state stores
2. Format a readable report for slash commands
3. Surface a compact status hint in the Pi footer

### Technical Approach

```ts
const rows = runtime.getStatusRows();
ctx.ui.setStatus("oauth-router", `${healthy}/${rows.length} healthy`);
```

### Key Considerations
- Output must stay readable with 2–10+ accounts
- Status must not expose access or refresh tokens
- Footer status should remain compact

## Acceptance Criteria

- [ ] `/router-status` shows overall policy and health
- [ ] `/router-accounts` shows per-account state
- [ ] Output includes enabled/disabled, cooldown, last used, failures, and expiry information
- [ ] Secret values are redacted or omitted
