# [FR-003] Credential Storage

## Labels
`MUS`, `storage`, `security`

## User Story
As a Pi user, I want my router accounts to persist across restarts, so that I do not need to log in again every time Pi starts.

## Proposed Solution

### Overview
Store credentials in an extension-specific secret file and keep runtime health state in a separate non-secret file.

### Implementation Flow
1. Create an extension data directory under `~/.pi/agent/oauth-router/`
2. Persist credentials to `credentials.json`
3. Persist health and routing state to `state.json`

### Technical Approach

```ts
interface StoredRouterAccount {
  id: string;
  provider: string;
  access: string;
  refresh: string;
  expires: number;
  enabled: boolean;
}
```

### Key Considerations
- Credentials must be clearly separated by account ID
- File permissions should be restrictive where supported
- Status output must redact secrets

## Acceptance Criteria

- [ ] Multiple accounts persist across Pi restarts
- [ ] Credentials are stored outside Pi’s single-provider auth slot
- [ ] Access and refresh token metadata are stored per account
- [ ] Health state is stored separately from secret material
