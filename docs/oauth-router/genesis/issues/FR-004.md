# [FR-004] Routing Policy

## Labels
`MUS`, `routing`, `reliability`

## User Story
As a Pi user, I want the router to prefer healthy accounts automatically, so that temporary rate limits or failures do not stop my workflow.

## Proposed Solution

### Overview
Implement round robin and weighted round robin selection with health checks, 429 cooldowns, disabled-account skipping, and transient penalties.

### Implementation Flow
1. Build the eligible account list for the requested model
2. Apply the selected routing policy
3. Update persistent state after each success or failure

### Technical Approach

```ts
const eligible = runtime.getEligibleAccounts(modelId);
const picked = chooseEligibleAccount(policy, eligible, cursor);
```

### Key Considerations
- Cooldowns must respect `Retry-After` when available
- Auth failures should quarantine an account until re-auth or manual recovery
- Weighted routing should still behave deterministically for small account pools

## Acceptance Criteria

- [ ] Round robin is supported
- [ ] Weighted round robin is supported
- [ ] Disabled accounts are skipped
- [ ] 429 responses trigger cooldowns
- [ ] 5xx/network failures trigger temporary penalties
- [ ] Auth failures prevent reuse until recovered
