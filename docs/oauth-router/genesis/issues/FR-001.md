# [FR-001] Custom oauth-router Provider

## Labels
`MUS`, `provider`, `pi-extension`

## User Story
As a Pi user, I want to select `oauth-router/<model>`, so that Pi routes requests automatically across configured accounts.

## Proposed Solution

### Overview
Register a custom provider named `oauth-router` during extension load and expose a configurable model catalog that Pi can use like any normal provider.

### Implementation Flow
1. Register the provider from the auto-loaded extension entrypoint
2. Load model and upstream config from the extension-owned config file
3. Delegate requests to the selected upstream implementation at stream time

### Technical Approach

```ts
pi.registerProvider("oauth-router", {
  api: "oauth-router-api",
  models: runtime.getProviderModels(),
  streamSimple: (model, context, options) => runtime.stream(model, context, options),
});
```

### Key Considerations
- The provider must auto-load from Pi’s global extension directory
- Provider registration must not rely on a separate daemon
- Model IDs should remain stable across restarts

## Acceptance Criteria

- [ ] `oauth-router` appears in Pi model listings
- [ ] Provider exposes configured model entries
- [ ] Selecting `oauth-router/<model>` uses router-controlled request execution
- [ ] No external process is required for routing
