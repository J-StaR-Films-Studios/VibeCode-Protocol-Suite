# FR-006: Compact Takomi Runtime and Policy Packs

## Labels

takomi-context-manager, mus

## User Story

As a Pi/Takomi user or agent, I want compact takomi runtime and policy packs, so that context is loaded only when useful without losing capability.

## Proposed Solution

Keep Takomi state compact and lazy-load heavy policies.

## Implementation Flow

- Represent gates as structured compact state
- Move full model routing to policy pack
- Load subagent/model policy together before delegation

## Technical Approach

Use a Pi TypeScript extension where possible. Prefer `before_agent_start` for prompt shaping, registered tools for progressive skill access, and diagnostics for observability. Avoid forking Pi internals unless extension hooks prove insufficient.

## Key Considerations

- Preserve strong tool guidance.
- Keep always-on context minimal.
- Make behavior observable and testable.
- Coexist with Pi/Takomi current resource loading where possible.

## Acceptance Criteria

- [ ] Small tasks are not over-orchestrated
- [ ] Policy pack is available for delegation
- [ ] Workflow packs load only when selected

## Verification Notes

Test using `powershell -ExecutionPolicy Bypass -File .\scripts\pi-dev.ps1` and/or a global `pi` invocation outside this repository.
