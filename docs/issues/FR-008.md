# FR-008: Orchestration Task Quality Gate

## Labels

takomi-context-manager, mus

## User Story

As a Pi/Takomi user or agent, I want orchestration task quality gate, so that context is loaded only when useful without losing capability.

## Proposed Solution

Prevent broad projects from becoming one vague orchestration task.

## Implementation Flow

- Define decomposition heuristics
- Add checklist for task quality
- Surface warning when scope is broad but task count/detail is low

## Technical Approach

Use a Pi TypeScript extension where possible. Prefer `before_agent_start` for prompt shaping, registered tools for progressive skill access, and diagnostics for observability. Avoid forking Pi internals unless extension hooks prove insufficient.

## Key Considerations

- Preserve strong tool guidance.
- Keep always-on context minimal.
- Make behavior observable and testable.
- Coexist with Pi/Takomi current resource loading where possible.

## Acceptance Criteria

- [ ] Broad projects receive multiple meaningful tasks
- [ ] Single-task Genesis has justification
- [ ] Quality checklist is reusable

## Verification Notes

Test using `powershell -ExecutionPolicy Bypass -File .\scripts\pi-dev.ps1` and/or a global `pi` invocation outside this repository.
