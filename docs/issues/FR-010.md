# FR-010: Semantic/RAG Skill Matching

## Labels

takomi-context-manager, future

## User Story

As a Pi/Takomi user or agent, I want semantic/rag skill matching, so that context is loaded only when useful without losing capability.

## Proposed Solution

Use semantic matching for vague context requests.

## Implementation Flow

- Add embedding or fuzzy scoring adapter
- Use recent conversation summary
- Tune thresholds

## Technical Approach

Use a Pi TypeScript extension where possible. Prefer `before_agent_start` for prompt shaping, registered tools for progressive skill access, and diagnostics for observability. Avoid forking Pi internals unless extension hooks prove insufficient.

## Key Considerations

- Preserve strong tool guidance.
- Keep always-on context minimal.
- Make behavior observable and testable.
- Coexist with Pi/Takomi current resource loading where possible.

## Acceptance Criteria

- [ ] Vague prompts surface relevant skills
- [ ] False positives remain low
- [ ] Can be disabled

## Verification Notes

Test using `powershell -ExecutionPolicy Bypass -File .\scripts\pi-dev.ps1` and/or a global `pi` invocation outside this repository.
