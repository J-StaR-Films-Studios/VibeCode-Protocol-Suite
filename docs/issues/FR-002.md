# FR-002: Skill Index

## Labels

takomi-context-manager, mus

## User Story

As a Pi/Takomi user or agent, I want skill index, so that context is loaded only when useful without losing capability.

## Proposed Solution

Expose available skills by name only by default.

## Implementation Flow

- Discover loaded skills
- Render names-only index
- Avoid descriptions/locations in always-on block

## Technical Approach

Use a Pi TypeScript extension where possible. Prefer `before_agent_start` for prompt shaping, registered tools for progressive skill access, and diagnostics for observability. Avoid forking Pi internals unless extension hooks prove insufficient.

## Key Considerations

- Preserve strong tool guidance.
- Keep always-on context minimal.
- Make behavior observable and testable.
- Coexist with Pi/Takomi current resource loading where possible.

## Acceptance Criteria

- [x] Agent sees skill names
- [x] Descriptions are not always injected
- [x] Index updates after reload

## Verification Notes

Test using `powershell -ExecutionPolicy Bypass -File .\scripts\pi-dev.ps1` and/or a global `pi` invocation outside this repository.
