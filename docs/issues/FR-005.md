# FR-005: Context Router Candidate Surfacing

## Labels

takomi-context-manager, mus

## User Story

As a Pi/Takomi user or agent, I want context router candidate surfacing, so that context is loaded only when useful without losing capability.

## Proposed Solution

Suggest likely context packs without injecting full content.

## Implementation Flow

- Score prompt against skill names/descriptions
- Inject top candidates only
- Record reasons for diagnostics

## Technical Approach

Use a Pi TypeScript extension where possible. Prefer `before_agent_start` for prompt shaping, registered tools for progressive skill access, and diagnostics for observability. Avoid forking Pi internals unless extension hooks prove insufficient.

## Key Considerations

- Preserve strong tool guidance.
- Keep always-on context minimal.
- Make behavior observable and testable.
- Coexist with Pi/Takomi current resource loading where possible.

## Acceptance Criteria

- [ ] Vague prompts get relevant candidates
- [ ] Low-confidence prompts do not add noise
- [ ] Candidates are names plus action hint only

## Verification Notes

Test using `powershell -ExecutionPolicy Bypass -File .\scripts\pi-dev.ps1` and/or a global `pi` invocation outside this repository.
