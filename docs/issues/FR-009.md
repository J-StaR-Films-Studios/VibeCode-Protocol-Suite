# FR-009: Duplicate Extension Conflict Detection

## Labels

takomi-context-manager, future

## User Story

As a Pi/Takomi user or agent, I want duplicate extension conflict detection, so that context is loaded only when useful without losing capability.

## Proposed Solution

Detect or mitigate duplicate global/project Takomi extension tool conflicts.

## Implementation Flow

- Inspect active extension/tool names if possible
- Warn about duplicate registrations
- Document remediation

## Technical Approach

Use a Pi TypeScript extension where possible. Prefer `before_agent_start` for prompt shaping, registered tools for progressive skill access, and diagnostics for observability. Avoid forking Pi internals unless extension hooks prove insufficient.

## Key Considerations

- Preserve strong tool guidance.
- Keep always-on context minimal.
- Make behavior observable and testable.
- Coexist with Pi/Takomi current resource loading where possible.

## Acceptance Criteria

- [ ] Conflict is visible before dispatch
- [ ] User gets clear fix path
- [ ] No silent subagent failure

## Verification Notes

Test using `powershell -ExecutionPolicy Bypass -File .\scripts\pi-dev.ps1` and/or a global `pi` invocation outside this repository.
