# FR-001: Prompt Kernel and Tool Contract

## Labels

takomi-context-manager, mus

## User Story

As a Pi/Takomi user or agent, I want prompt kernel and tool contract, so that context is loaded only when useful without losing capability.

## Proposed Solution

Keep a compact always-on prompt while preserving precise tool guidance.

## Implementation Flow

- Compact identity and behavior rules
- Precise read/bash/edit/write guidance
- Anti-ceremony task sizing rule

## Technical Approach

Use a Pi TypeScript extension where possible. Prefer `before_agent_start` for prompt shaping, registered tools for progressive skill access, and diagnostics for observability. Avoid forking Pi internals unless extension hooks prove insufficient.

## Key Considerations

- Preserve strong tool guidance.
- Keep always-on context minimal.
- Make behavior observable and testable.
- Coexist with Pi/Takomi current resource loading where possible.

## Acceptance Criteria

- [ ] Prompt kernel exists
- [ ] Tool contract remains actionable
- [ ] No full skill descriptions in kernel

## Verification Notes

Test using `powershell -ExecutionPolicy Bypass -File .\scripts\pi-dev.ps1` and/or a global `pi` invocation outside this repository.
