These HTML mockups test the design direction recommended by the Takomi designer on Gemini 3.1: a bottom card stack that makes delegation feel like a visible call stack near the command line, instead of a detached side panel. The goal is to make nested delegation, steering, and baton-passing feel obvious.

Why this works better

  • Eyes stay anchored near the input where interaction actually happens.
  • Delegation depth is obvious because cards stack like a call stack.
  • Steering feels native because the active card becomes the input target.
  • The transcript stays clean while active state remains visible.

Terminal adaptation rules

RunningBright active card at the bottom, parent cards dimmed above it.
CompleteChild folds into parent, then the summary drops into transcript history.
BlockedCard stays open and input automatically targets the blocked agent for rescue.
Key pattern Transcript = history
Bottom stack = now
Steer mode copy @designer > also keep logs condensed
Nested baton pass Parent card dims → child card appears underneath → completion folds upward.
